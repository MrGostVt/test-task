# Description

Старався зробити схожим на google forms, але через брак часу не встигаю реалізувати ще декілька систем:
 - RequiredQuestions: Обов'язкові запитання
 - Grades: Система оцінювання, де під час створення форми користувач вводить кількість балів за правильну відповідь.
 - InputValidation: Система валідації вводу на клієнті, базова валідація через regExp та екранування. В API є базова валідація, але вона не дороблена.
 - Системи пагінації для оптимізації отримання форм та відповідей на них + віртуальні списки як розширення оптимізації на клієнті.

GraphQL 
Реалізував всі зазначенні endpoint, окрім responces, оскільки вважаю що він не доречний, оскільки його можна замінити на form з розширеним запитом данних. Плюс з цим зв`язана модифікація роуту на клієнті, де відповіді виведені на підсторінку, окремо від списку відповідей, в якому тепер знаходиться посилання.

SharedTypes
Я до кінця не встиг розібратись з правильним налаштуванням репозиторію (до цього не користувався monorepo підходом), тому вирішив поки що скопіювати спільні типи.
